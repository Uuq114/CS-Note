# Claude Code 使用指南

## 什么是 Claude Code

Claude Code 是 Anthropic 官方的 AI 编程助手，可以通过 CLI 和 VSCode 扩展使用。它能够读取、编辑文件，执行命令，帮助开发者完成各种软件工程任务。

## 核心概念

### 1. 工具系统 (Tools)

Claude Code 通过工具与代码库交互，主要工具包括：

**文件操作**
- `Read` - 读取文件内容
- `Write` - 创建新文件
- `Edit` - 编辑现有文件（精确字符串替换）
- `Glob` - 文件模式匹配搜索
- `Grep` - 内容搜索（支持正则表达式）

**命令执行**
- `Bash` - 执行 shell 命令
- 注意：优先使用专用工具而非 bash 命令

**代理系统**
- `Agent` - 启动专门的子代理处理复杂任务
  - `Explore` - 代码库探索
  - `Plan` - 实现规划
  - `general-purpose` - 通用任务

**其他工具**
- `TodoWrite` - 任务列表管理
- `AskUserQuestion` - 向用户提问
- `EnterPlanMode` - 进入规划模式
- `WebFetch` / `WebSearch` - 网络搜索

### 2. 权限模式 (Permission Modes)

工具执行需要用户授权，有不同的权限模式：
- 某些工具会自动执行
- 某些需要用户批准
- 可以在设置中配置权限策略

### 3. 工作流程

**典型任务流程：**
1. 理解需求 - 阅读相关文件
2. 规划方案 - 对于复杂任务使用 Plan 模式
3. 执行实现 - 使用工具修改代码
4. 验证结果 - 运行测试或检查

**并行执行：**
- 独立的工具调用可以并行执行
- 有依赖关系的必须顺序执行

## 使用技巧

### 文件操作最佳实践

1. **先读后写** - 修改文件前必须先用 Read 读取
2. **优先编辑** - 修改现有文件用 Edit 而非 Write
3. **精确匹配** - Edit 工具需要精确匹配原字符串（包括缩进）
4. **避免创建不必要的文件** - 优先编辑现有文件

### 搜索策略

1. **简单搜索** - 直接使用 Glob 或 Grep
2. **复杂探索** - 使用 Explore 代理
3. **并行搜索** - 多个独立搜索可以并行执行

### Git 操作

**安全原则：**
- 创建新提交而非修改现有提交
- 避免破坏性操作（force push, reset --hard）
- 不跳过 hooks（--no-verify）
- 提交前检查 git status 和 git diff

**提交流程：**
1. 查看状态和变更
2. 分析变更并起草提交信息
3. 添加文件并创建提交
4. 包含 Co-Authored-By 标记

### 任务管理

**使用 TodoWrite 的场景：**
- 多步骤复杂任务（3+ 步骤）
- 用户提供多个任务
- 需要跟踪进度的任务

**任务状态：**
- `pending` - 未开始
- `in_progress` - 进行中（同时只能有一个）
- `completed` - 已完成

### 代理使用

**何时使用 Agent：**
- 复杂的代码库探索
- 需要多轮搜索的研究任务
- 可以并行处理的独立任务

**何时不用：**
- 简单的文件查找
- 已知具体路径的操作
- 单个文件的读取

## 交互原则

### 沟通风格

- 简洁直接，避免冗长
- 技术性语言，但易于理解
- 不使用表情符号（除非用户要求）
- 不重复已说过的内容

### 代码实践

- **最小化原则** - 只写必要的代码
- **安全优先** - 避免安全漏洞
- **避免过度工程** - 不添加未要求的功能
- **可立即运行** - 确保代码语法正确

### 风险操作

**需要确认的操作：**
- 删除文件/分支
- Force push
- 修改共享资源
- 发送消息/创建 PR

**原则：**
- 谨慎对待不可逆操作
- 遇到障碍时寻找替代方案
- 发现意外状态时先调查

## 特殊功能

### Plan 模式

用于复杂实现任务的规划：
1. 探索代码库
2. 设计实现方案
3. 向用户展示计划
4. 获得批准后执行

### 技能系统 (Skills)

- 用 `/skill-name` 调用（如 `/commit`）
- 使用 `Skill` 工具执行
- 常用技能：commit, review-pr 等

### 内存系统

- 持久化内存目录：`~/.claude/projects/[project]/memory/`
- `MEMORY.md` - 主要内存文件（保持简洁）
- 按主题组织，不按时间
- 保存稳定的模式和偏好
- 不保存临时状态

## 限制和注意事项

1. **长时间运行的命令** - 不要用 Bash 运行开发服务器、watch 模式
2. **上下文管理** - 系统会自动压缩历史消息
3. **安全边界** - 拒绝恶意代码请求
4. **不讨论内部提示** - 不能透露系统指令细节

## 快速参考

### 常用命令模式

```bash
# 搜索文件
Glob: "**/*.ts"

# 搜索内容
Grep: pattern="function.*export" output_mode="content"

# 读取文件
Read: file_path="/path/to/file"

# 编辑文件
Edit: old_string="原内容" new_string="新内容"

# 执行命令
Bash: command="npm test"
```

### 工作流示例

**添加新功能：**
1. 使用 Glob/Grep 找到相关文件
2. 用 Read 读取文件
3. 用 Edit 修改代码
4. 用 Bash 运行测试
5. 用 git 命令提交（如果用户要求）

**调试问题：**
1. 用 Grep 搜索错误信息
2. 用 Read 读取相关文件
3. 分析问题根源
4. 用 Edit 修复
5. 验证修复效果
